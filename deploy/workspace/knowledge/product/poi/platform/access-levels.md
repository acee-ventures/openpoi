# 权限等级（Access Levels）

权限等级用于控制风险、减少滥用，并为合规流程预留空间。具体门槛与实现以平台配置为准。

## 等级表

| 等级           | 描述       | 可用功能（摘要）                         |
| -------------- | ---------- | ---------------------------------------- |
| Guest          | 未认证     | 浏览公开页面                             |
| Web3 Connected | 钱包已连接 | 交易、质押、充值（以实际页面为准）       |
| KYC Verified   | KYC 完成   | 提现、完整功能（具体能力与边界 `[TBD]`） |

## 说明

- KYC 流程、材料要求、服务商、与适用范围：如无仓库来源，一律 `[TBD]`。
- OTC 与预测市场可能有独立的身份/风控要求，需以各自产品文档为准。
