# OTC 使用指南（操作员视角）

本指南面向 OTC 操作员，描述客户注册、报价、交易执行、TRON 转账确认与合规留痕的标准流程。
注意：本指南为流程型说明，具体政策参数（限额、费率、材料清单）如无仓库来源一律 `[TBD]`。

## 客户注册与 KYC（进行中 / 规划中）

1. 创建客户档案：姓名、联系方式、基础标识信息。
2. 收集 KYC 材料：材料清单与校验规则 `[TBD]`。
3. 设置 KYC 状态：pending → verified / rejected。
4. 设置交易限额与风险级别：按内部政策 `[TBD]`。

## 获取报价与锁价

1. 确认方向：客户买入 USDT / 卖出 USDT。
2. 录入金额与网络：默认 TRC20。
3. 生成报价并锁定有效期（有效期 `[TBD]`）。
4. 将报价信息记录到交易备注（便于审计）。

## 创建交易

1. 创建交易记录：客户、方向、金额、支付方式、网络等信息。
2. 核对客户钱包地址与网络（TRC20）。
3. 生成内部参考号与凭证留痕字段（receipt/notes）。

## TRON 转账与确认

1. 执行 TRON 链转账（TRC20 USDT）。
2. 记录 txHash，并等待确认（确认数策略 `[TBD]`）。
3. 更新交易状态：created → pending_chain → completed（状态枚举以系统为准）。

## C2C 点对点担保（概览）

1. 确认买卖双方均为 verified 客户。
2. 卖家发布卖单并完成担保入金确认。
3. 买家创建 trade 锁定卖单，并在时限内完成线下/法币支付（时限 `[TBD]`）。
4. 买家标记已付款并上传凭证（如启用）；卖家确认收款后释放担保资产。
5. 如发生争议：进入 dispute 流程并由管理员按政策处置（政策 `[TBD]`）。

## 合规留痕（进行中 / 规划中）

- 记录保留：目标 5 年（具体实现与存储策略 `[TBD]`）。
- CTR/SAR：符合触发条件时生成内部 case 并升级处理（触发规则 `[TBD]`）。
